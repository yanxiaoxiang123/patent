# 四个审核智能体实现文档

## 概述

智能专利辅助审核系统（IPRS）包含四个审核智能体，基于 Ollama 本地大语言模型（qwen3:8b）实现，通过规则模板 + LLM 的方式完成专利文档审核。

---

## 四个审核智能体

| template_id | 名称 | 规则文件 | 功能定位 |
|------------|------|---------|---------|
| 1 | 普通案例审核专家 | `general_case_rules.json` | 普通专利申请案件审核 |
| 2 | 专利审核指导 | `patent_guidance_rules.json` | 专利审核指导与学习 |
| 3 | 专案案例审核专家 | `project_case_rules.json` | 专案/优审案件审核（更严格） |
| 5 | IPC 分类专家 | `ipc_classification_rules.json` | 国际专利分类号分析 |

---


## 核心文件说明

### 1. chat.py (API 入口)

**路径**: `backend/app/api/chat.py`

**功能**:
- 提供 `/api/chat` 接口
- 识别 template_id 参数
- 构建消息并调用 Ollama
- 返回 SSE 流式响应

**关键代码**:

```python
# 加载严格模板
if chat_request.template_id in [1, 2, 3, 5]:
    strict_system_prompt = await load_strict_template(chat_request.template_id)

# Passthrough 模式使用审核规则
if passthrough:
    return EventSourceResponse(
        create_stream_generator(..., strict_system_prompt),
        headers=SSE_HEADERS
    )
```

### 2. loader.py (规则加载器)

**路径**: `backend/app/prompts/rules/loader.py`

**功能**:
- 加载 JSON 格式的审核规则
- 将规则格式化为 system prompt
- 管理规则缓存

**规则文件映射**:

```python
rule_files = {
    1: "general_case_rules.json",       # 普通案例审核
    2: "patent_guidance_rules.json",    # 专利审核指导
    3: "project_case_rules.json",       # 专案案例审核
    5: "ipc_classification_rules.json", # IPC 分类指导
}
```

### 3. ollama.py (Ollama 服务封装)

**路径**: `backend/app/services/ollama.py`

**功能**:
- 调用本地 Ollama API
- 支持流式/非流式响应
- 处理 thinking 思考过程
- 管理模型配置

## 审核智能体详细说明

### template_id = 1: 普通案例审核专家

**规则文件**: `backend/app/prompts/rules/general_case_rules.json`

**审核范围**: 适用于一般专利代理案件

**输出结构**:

```
① 案件类型判定
   - 判定结果：实用新型 / 普通发明
   - 判定依据（引用原文条款）

② 通用规则预审（A-F）
   A. 格式规范
      - 编号统一
      - 引用规范
      - 附图标记
      - 标题用语
   B. 结构完整性
      - 技术领域
      - 背景技术
      - 发明内容
      - 附图说明
      - 具体实施方式
   C. 逻辑一致性
   D. 语言表达
   E. 风险点
   F. 排除项

③ 分类型审核清单
   - 格式审查（✅/❌/信息缺失）
   - 内容审查（含新颖性/创造性）

④ 结论与建议
   - 审核结论
   - 修改优先级
   - 需补充材料清单
```

**判断标准**:

| 案件类型 | 判断标准 |
|---------|---------|
| 实用新型 | 权利要求均为产品结构特征描述，说明书仅描述产品结构及其组装方式，未提及制造方法、工艺步骤 |
| 普通发明 | 权利要求包含方法特征，或说明书描述了制造方法、使用工艺步骤，或涉及方法改进 |

---

### template_id = 2: 专利审核指导

**规则文件**: `backend/app/prompts/rules/patent_guidance_rules.json`

**审核范围**: 专利审核指导与学习

**输出结构**:

```
一、文档整体概览
   - 技术领域识别
   - 发明类型判断
   - 保护范围分析

二、关键技术特征和创新点
   - 独立权利要求特征
   - 从属权利要求特征
   - 创新点提炼

三、格式规范指导
   A. 权利要求书规范
   B. 说明书规范
   C. 附图规范

四、内容质量指导
   - 清楚
   - 完整
   - 支持
   - 简要

五、专利审核知识点
   - 相关法条
   - 审查标准
   - 常见问题
```

---

### template_id = 3: 专案案例审核专家

**规则文件**: `backend/app/prompts/rules/project_case_rules.json`

**审核范围**: 专案/优审案件（要求更严格）

**特点**:
- 通用规则预审要求更严格
- 增加领域审查（IPC分类、快速通道判断）
- 格式审查为硬指标
- 内容审查更注重新颖性/创造性

**输出结构**:

```
① 通用规则预审（A-F）
   - 格式规范（更严格）
   - 结构完整性（更严格）
   - 逻辑一致性（更严格）
   - 语言表达（更严格）
   - 风险点（更严格）
   - 排除项（更严格）

② 领域审查
   - IPC分类检查
   - 快速通道判断
   - 专案修改建议

③ 格式审查（专案硬指标）
   - ✅/❌/信息缺失
   - 权利要求书硬性要求
   - 说明书硬性要求

④ 内容审查（含新颖性/创造性）
   - 新颖性分析
   - 创造性分析
   - 从权审查

⑤ 结论与建议
   - 审核结论
   - 修改优先级
   - 需补充材料清单
```

---

### template_id = 5: IPC 分类专家

**规则文件**: `backend/app/prompts/rules/ipc_classification_rules.json`

**审核范围**: 国际专利分类号（IPC）分析

**特点**:
- 仅输出 IPC 分类相关结论
- 不输出新颖性/创造性等可专利性分析
- 不输出撰写建议、风险提示

**输出结构**:

```
一、技术方案分析
   - 技术领域识别
   - 技术方案提取
   - 关键技术特征

二、主分类号分析
   - 大类判断
   - 小类判断
   - 组/小组判断
   - IPC 分类参考

三、辅分类号分析
   - 相关分类号
   - 分类依据

四、分类选择理由
   - 分类合理性
   - 分类依据说明
```

**IPC 分类参考示例**:

```
G06F 16/953 - 数字信息检索
G06F 16/958 - 内容索引
G06F 16/987 - 数据库结构
```

## 系统提示词构建

### format_rules_as_prompt() 函数

将 JSON 规则格式化为 LLM 可理解的 system prompt。

**构建流程**:

```
JSON 规则
    │
    ▼
提取 template_name → 角色定义
    │
    ▼
添加重要要求（引用原文、具体位置、可操作建议）
    │
    ▼
按规则类型逐项格式化
    │
    ▼
添加输出格式要求
    │
    ▼
返回完整 system prompt
```

**示例输出**:

```
你是资深普通案例审核专家，对专利申请书进行审核并输出详细、结构化的审核报告。

【重要要求】
- 每个审核项必须给出详细、具体的分析，篇幅充分
- 问题描述要引用原文，说明具体位置（章/节/段落/权利要求号）
- 修改建议要具体、可操作，不能泛泛而谈
- 对于通过的项目，也要简要说明检查过程和结论依据

【案件类型判定】
根据文本判断：实用新型 / 普通发明
输出：判定结果 + 详细依据（必须引用原文条款，说明判断依据）

判断标准：
- 实用新型：权利要求均为产品结构特征描述...
- 普通发明：权利要求包含方法特征...
```

## Ollama 调用配置

**模型**: qwen3:8b

**API 端点**: `/api/chat`

**请求格式**:

```python
payload = {
    "model": "qwen3:8b",
    "messages": [
        {"role": "system", "content": "<审核规则 prompt>"},
        {"role": "user", "content": "<专利文档内容>"}
    ],
    "think": True,  # 启用思考过程
    "stream": True,
    "options": {
        "temperature": 0.3,  # 审核模式使用低温
        "top_p": 0.8,
        "num_ctx": 32768,
        "repeat_penalty": 1.1,  # 审核模式使用重复惩罚
        "max_tokens": 40960,
        "keep_alive": "24h"
    }
}
```

**响应格式**:

```json
{
    "message": {
        "role": "assistant",
        "content": "审核报告内容...",
        "thinking": "思考过程..."
    }
}
```

## 扩展方向

### 当前实现的局限性

1. **单轮对话** - 一次性生成审核结果，无多步骤规划
2. **无工具调用** - 不能搜索专利库、查法条
3. **无反思机制** - 不能自我修正审核结果
4. **规则静态** - JSON 规则需要手动更新

## 使用示例

### 请求

```bash
curl -X POST http://localhost:8000/api/chat \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{
    "messages": [
      {"role": "user", "content": "专利文档内容..."}
    ],
    "stream": true,
    "template_id": 1,
    "model": "qwen3:8b"
  }'
```

### 响应 (SSE 流式)

```
data: {"choices": [{"delta": {"thinking": "正在分析文档结构..."}}]}

data: {"choices": [{"delta": {"content": "① 案件类型判定"}}]}

data: {"choices": [{"delta": {"content": "根据权利要求分析..."}}]}

...

data: [DONE]
```

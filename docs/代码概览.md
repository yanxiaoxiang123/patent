# 智能专利辅助审核系统（IPRS）代码概览

本文件总结仓库的整体功能、架构与技术栈，并给出关键模块的职责、主要 API、运行与部署要点以及源码定位链接，便于快速上手与维护。

## 项目摘要

- 用途：面向专利代理与审核场景，支持文档上传与解析、AI 审核与对话、会话持久化与管理员用户管理。
- 架构：后端 FastAPI + MySQL（SQLAlchemy 异步）、前端 Vue 3（Element Plus/Ant Design Vue/Ant Design X Vue），AI 走本地 Ollama（支持 SSE 流式与 think）。
- 交互：单页聊天视图，附件上传解析预览，流式展示“思考过程/最终回答”，会话列表管理，管理员用户 CRUD。

## 端到端业务流

- 登录与权限
  - 前端保存 Bearer Token（localStorage），路由守卫校验；后端解析 simple_token_{id}_{username} 并提供管理员接口。
- 文档解析
  - 上传文件→创建 DB 记录（uploaded）→解析服务抽取结构化 JSON→更新状态（parsed）→前端节选展示与预览。
- 对话与会话
  - 构建系统与用户消息→后端调用 Ollama（/api/chat 或 /api/generate）→SSE 推送 delta.thinking/content→前端实时渲染并持久化会话与消息→侧栏管理/续写/删除。

## 主要 API（后端）

- 健康检查
  - GET `/`、GET `/health`（[main.py](file:///www/wwwroot/iprs/patent/backend/app/main.py)）
- 认证与管理员
  - POST `/api/auth/login`、GET `/api/auth/me`、POST `/api/auth/logout`（[main.py](file:///www/wwwroot/iprs/patent/backend/app/main.py)）
  - GET/POST/DELETE `/api/admin/users[/id]`（管理员权限）（[main.py](file:///www/wwwroot/iprs/patent/backend/app/main.py)）
- 文档管理（prefix `/api/documents`，详见 [documents.py](file:///www/wwwroot/iprs/patent/backend/app/api/documents.py)）
  - POST `/upload`、GET `/`、GET `/{id}`、POST `/{id}/parse`、DELETE `/{id}`、GET `/{id}/download`
- AI 对话（prefix `/api/ai`，详见 [chat.py](file:///www/wwwroot/iprs/patent/backend/app/api/chat.py)）
  - POST `/chat`（支持模板严格模式、passthrough、SSE）
  - GET `/sessions`、GET `/sessions/{id}`、DELETE `/sessions/{id}`
  - GET `/models`（Ollama tags）、GET `/status`（健康）
  - POST `/warm`（模型预热）

## 流式（SSE）实现要点

- 后端
  - 将 Ollama 逐行 JSON 流统一转为 OpenAI-like 片段：`choices[0].delta.thinking/content`，完成时发送 `[DONE]`。
  - 模板严格模式：起始标记捕捉与“修复重排”，必要时通过替换提示控制输出格式。
- 前端
  - 以 fetch 解析 `text/event-stream`，拼接 thinking 与 content，并在输出首段插入“思考过程/最终回答”标签；见 [SimplePatentChat.vue](file:///www/wwwroot/iprs/patent/frontend/src/views/SimplePatentChat.vue#L1020-L1146)。

## 约定与注意事项

- 简化 Bearer Token：`simple_token_{id}_{username}`，后端在请求头解析当前用户与管理员权限。
- 文档解析需正确配置上传目录与系统工具（针对旧 `.doc`）。
- AI 模型需在本地 Ollama 预先拉取，如 `qwen3:8b`；支持 `think: true` 与 streaming。
- 前后端跨域需根据 CORS 与代理配置调整。

## 已知问题与改进方向

- chat 路由中 `list_models` 与 `get_ai_status` 的 `OLLAMA_URL` 变量需与多实例轮询逻辑统一（当前文件顶部使用 `OLLAMA_URLS`）。
- 提示词模板可扩展 IPC 分类严格模式与更多审核类型。
- 文档解析旧 `.doc` 的外部依赖可增加可用性检查与降级策略提示。

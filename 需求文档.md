产品需求文档 (PRD): 智能专利辅助审核系统 (IPRS)文档版本修改日期修改人备注v1.02025-12-13Gemini初始版本创建1. 项目概述 (Project Overview)1.1 背景当前专利代理行业面临人工审核效率低、低级错误（错别字、引用错误）排查耗时、逻辑一致性检查困难等痛点。本项目旨在利用大语言模型技术，构建一套 Web 端智能审核系统。1.2 核心目标效率提升： 将形式审查时间从平均 30 分钟缩短至 1 分钟。质量保障： 利用 AI 辅助发现逻辑漏洞和新颖性风险。技术融合： 结合 本地 Ollama (Qwen3-7b) 的低延迟优势与 云端 Coze Agent 的复杂推理能力。1.3 用户画像核心用户： 专利代理人/撰写工程师（如“李专员”）。需求特征： 重视数据隐私，追求高准确率，习惯 Word 批注式工作流。2. 功能需求说明 (Functional Requirements)优先级定义：P0 (必须有), P1 (应该有), P2 (可以有)2.1 模块一：账户与项目管理 (Auth & Workspace)ID功能名称优先级详细描述验收标准 (Acceptance Criteria)F-1.1用户注册/登录P0基于 JWT 的身份验证体系。1. 支持账号密码登录。2. Token 过期需自动跳转登录页。F-1.2案件仪表盘P0展示用户所属的所有专利案件列表。1. 列表包含：案件号、案件名、状态（待审/已审）、创建时间。2. 点击列表项进入详情页。F-1.3状态流转P1案件状态自动更新机制。上传后为“待解析” -> 解析后为“待审核” -> 审核中 -> “已完成”。2.2 模块二：文档解析与预处理 (Ingestion & Parsing)ID功能名称优先级详细描述验收标准 (Acceptance Criteria)F-2.1文件上传P0支持 .docx 和 .pdf 格式上传。1. 单个文件大小限制 20MB。2. 上传显示进度条。F-2.2智能分段解析P0(核心算法) 后端需识别文档结构，提取关键字段。1. 系统能自动将文本拆分为：[摘要], [权利要求书], [说明书] 三大块。2. 解析失败需报错提示。F-2.3文本编辑器P0左侧只读/可编辑的文本展示区。1. 保持原文段落结构。2. 支持行号显示，便于定位。2.3 模块三：AI 智能审核 (AI Core Review)此模块采用 双模型策略。ID功能名称模型来源优先级详细描述验收标准F-3.1形式错误扫描Ollama (Qwen3-7b)P0快速检查错别字、标点、附图标记错误。1. 响应时间 < 5秒/页。2. 错误处在编辑器中高亮显示（如黄色下划线）。3. 鼠标悬停显示修正建议。F-3.2权利要求支持性分析Coze AgentP0检查《权利要求书》中的技术特征是否在《说明书》中有对应描述。1. 侧边栏流式输出（Streaming）分析报告。2. 明确指出“缺乏支持”的权利要求编号。F-3.3术语一致性检查Coze AgentP1检查全文术语定义是否前后矛盾。1. 输出矛盾术语列表（例：“控制器” vs “控制单元”）。F-3.4IPC 分类推荐Coze AgentP2根据摘要推荐 IPC 分类号。1. 输出 Top-3 推荐分类号及其置信度。2.4 模块四：交互与修正 (Interaction)ID功能名称优先级详细描述验收标准F-4.1一键采纳P0用户点击 AI 建议，自动替换原文。1. 替换后高亮消失。2. 支持撤销操作 (Undo)。F-4.2智能问答 (Copilot)P1选中一段文本，向 AI 提问。1. 类似 Chat 界面。2. 上下文包含选中的文本段落。F-4.3报告导出P1生成审核报告。1. 格式支持 PDF。2. 报告包含：整体评分、主要风险点列表。3. 非功能需求 (Non-Functional Requirements)3.1 性能要求Ollama 响应： 形式审查 API 平均响应时间需控制在 200ms/token 以内。Coze 响应： 由于网络原因，Coze 可能较慢，前端必须实现 流式渲染 (Streaming)，避免用户以为系统卡死。3.2 安全与隐私数据隔离： 不同用户的专利数据必须严格物理或逻辑隔离。模型隐私： 使用 Ollama 本地部署是为了确保最敏感的原文不完全依赖公网传输（如架构允许，可设置纯本地模式）。3.3 兼容性推荐使用 Chrome, Edge 最新版浏览器访问。4. 数据交互与逻辑流程 (System Logic)4.1 核心数据实体 (Entity)User: 用户基础信息。Project (Patent): 对应一个上传的文件。AuditLog: 存储 AI 的审核记录（JSON 格式），结构如下：JSON{

4.2 AI 调用逻辑差异场景 A：改错字 (Ollama)Prompt: "你是一个专业的专利校对员。请找出以下文本中的错别字和格式错误，输出 JSON 格式，不要废话。"优势: 免费、极速。场景 B：查逻辑 (Coze)Prompt: "你是一个资深专利审查员。请阅读说明书（Context），判断权利要求 1（Query）是否得到了充分支持。请一步步思考（Chain of Thought）。"优势: 逻辑强、知识库广（可联网搜索现有技术）。5. UI/UX 布局示意 (Wireframe Concept)页面布局：三栏式左栏 (15%)： 导航与文件大纲[Dashboard][历史记录]-- 大纲 --摘要权利要求书说明书中栏 (55%)： 交互式编辑器显示解析后的文本。交互： 红色波浪线表示错误，点击弹出气泡卡片。右栏 (30%)： AI 助手与报告面板[形式审查结果] (列表展示)[深度分析] (对话框/流式文本)[导出按钮]6. 风险评估 (Risk Analysis)幻觉风险 (Hallucination): 大模型可能会编造不存在的法律条款。对策: 在界面显著位置标注“AI 意见仅供参考，请人工复核”。解析失败风险: 用户上传的 Word 文档格式千奇百怪。对策: 强制提供一个“标准模板”供用户下载，或在解析失败时提供纯文本手动粘贴模式。
